CACHE LIFECYCLE
===============

Cache Flow:
┌────────────────────────────────────────────────────────────────────┐
│                         Cache State                                │
└────────────────────────────────────────────────────────────────────┘

1. EMPTY STATE (First Visit)
   ┌──────────────────┐
   │ localStorage:    │
   │ [empty]          │
   └──────────────────┘
        │
        ▼ Fetch profile from DB
        │
   ┌──────────────────────────────────────┐
   │ localStorage:                        │
   │ {                                    │
   │   profile: { role, email, name }     │
   │   timestamp: 1234567890              │
   │   userId: "abc-123"                  │
   │ }                                    │
   └──────────────────────────────────────┘


2. VALID CACHE (Within 5 minutes)
   ┌──────────────────────────────────────┐
   │ Age: 2 minutes                       │
   │ Status: ✅ VALID                     │
   │ Action: Read from cache immediately  │
   └──────────────────────────────────────┘
        │
        ▼ Render UI instantly
        │
        ▼ Background refresh (optional)


3. EXPIRED CACHE (After 5 minutes)
   ┌──────────────────────────────────────┐
   │ Age: 6 minutes                       │
   │ Status: ❌ EXPIRED                   │
   │ Action: Delete & fetch fresh         │
   └──────────────────────────────────────┘
        │
        ▼ Clear localStorage
        │
        ▼ Fetch from DB
        │
        ▼ Cache new data


4. CACHE INVALIDATION

   Triggers:
   • User signs out ────────────► Clear cache
   • Different user ID ─────────► Clear cache
   • Manual refresh ───────────► Fetch fresh & update cache
   • Invalid JSON ─────────────► Clear cache
